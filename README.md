# The House Without Walls

*A vast hypertext labyrinth.*

The House Without Walls is a large, handcrafted, static hypertext labyrinth. It is not a traditional game, not a generative system, and not a puzzle meant to be solved.

It is a place to wander.

Readers move through rooms, corridors, gardens, staircases, libraries, mirrors, and strange spaces that follow symbolic and dreamlike logic rather than physical architecture. There are branching paths, loops, dead ends, hidden passages, and many ways forward.

There is no danger here.

With persistence, every wanderer eventually reaches a center.

There is not one center, but many.

Each reveals the same truth in a different way.

---

## Concept

The House Without Walls is inspired by:

* hypertext literature
* labyrinths and symbolic spaces
* contemplative and philosophical writing
* dream logic rather than physical geography

The project explores a simple idea:

A person wanders through a labyrinth long enough that the wandering itself becomes the lesson.

Readers will experience:

* curiosity
* disorientation
* recognition of patterns
* acceptance of wandering
* moments of quiet realization

Eventually they encounter a **center**, where the labyrinth dissolves into understanding.

Then they may begin again.

---

## Design Principles

### Local clarity, global mystery

Each page should be easy to understand.

But the reader should **never be able to see the whole structure** of the labyrinth.

There is no map.

There is no progress bar.

The shape of the labyrinth must be discovered through wandering.

---

### Many paths, eventual convergence

The labyrinth contains:

* branching paths
* loops
* recombinations
* reflective dead ends
* hidden shortcuts

Over time, seemingly unrelated routes converge.

With enough persistence, all paths lead toward a center.

---

### Multiple centers

There is no single final node.

Different centers may appear as:

* a garden
* a mirror
* an empty room
* a pool of still water
* an open sky

Each represents a different metaphor for the same realization.

---

### Static and authored

This project intentionally avoids dynamic generation.

The labyrinth is **handwritten and static**.

Every page exists because someone placed it there.

The site can theoretically be deployed entirely as static files (for example on Cloudflare Pages).

This constraint is intentional and important.

It ensures the labyrinth feels like a discovered artifact rather than a program.

---

## Writing Style

The prose should be:

* calm
* contemplative
* symbolic
* dreamlike
* minimal but evocative

Spaces should feel tangible and sensory:

* lantern light
* quiet corridors
* wind in gardens
* distant bells
* mirrors and still water

The labyrinth should feel strange but gentle.

---

## Project Structure

All content is written in **Markdown files**.

The labyrinth is composed of nodes.

Each node represents a location in the labyrinth and contains:

* a title
* descriptive prose
* a set of choices leading to other nodes

Nodes are authored in Markdown and processed by build tooling that generates a static site.

This approach allows the labyrinth to scale to hundreds or thousands of pages while remaining easy to write and maintain.

## What This Project Is Not

This project is intentionally **not**:

* a traditional choose-your-own-adventure game
* an RPG with inventory or combat
* a procedural or AI-generated narrative
* a puzzle with a single solution

The House Without Walls is a **literary labyrinth**.

The goal is not to solve it.

The goal is to walk it.
